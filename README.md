# Deep-Learning-vs-Classical-Machine-Learning-for-Log-Anomaly-Detection-A-SIEM-Style-Evaluation

This repository contains our undergraduate thesis, "Deep Learning vs Classical Machine Learning for Log Anomaly Detection on System Log Dataset: A SIEM-Style Evaluation."

The study compares four log anomaly detection models—LSTM, Autoencoder (AE), LSTM Autoencoder (LSTM-AE), and Random Forest (RF)—using the HDFS v1 dataset from LogHub. Beyond traditional metrics such as Precision, Recall, F1-Score, and ROC-AUC, the evaluation emphasizes SIEM-oriented metrics including False Positive Rate (FPR), Alert Volume, and operational efficiency.

Our findings show that Random Forest achieved the best overall performance, while LSTM-AE was the strongest deep learning approach. The work highlights the importance of evaluating anomaly detection models from an operational security perspective rather than relying solely on benchmark accuracy.
